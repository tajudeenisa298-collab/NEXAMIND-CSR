Database Schema
Overview
SupportFlow AI is designed around a relational database using Supabase PostgreSQL with pgvector
for semantic search.
The database has four primary responsibilities:
1. Store organizations and users.
2. Store conversations and support activity.
3. Store the AI knowledge base.
4. Store analytics and automation history.
The schema is designed to support multi-tenancy, ensuring every organization's data remains
completely isolated.
Database Design Principles
The schema should be:
• Normalized
• Scalable
• Secure
• Multi-tenant
• AI-friendly
• Audit-friendly
Every table must include:
• UUID primary key
• created_at
• updated_at
• organization_id (where applicable)

Row Level Security (Supabase)
Every table must enforce organization isolation.
Example policy:
User may only access rows where
organization_id = current_user.organization_id
Customers
Only own conversations.
Agents
Only assigned tickets.
Managers
Entire organization.
Admins
Everything.
